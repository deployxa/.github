# Security Policy

## Supported Versions

Deployxa releases security updates for active production builds of `@deployxa/cli`, `deployxa-engine`, and `deployxa-dashboard`.

| Version | Supported |
| ------- | --------- |
| v1.10.x | ✅ Yes    |
| v1.0.x  | ⚠️ Legacy  |
| < v1.0  | ❌ No     |

---

## Reporting a Vulnerability

Security is a fundamental priority for Deployxa. If you discover a potential security vulnerability in any Deployxa repository, CLI tool, or cloud endpoint, please report it responsibly:

### How to Report
- **Email**: Send vulnerability reports directly to `security@deployxa.com`.
- **Encryption**: You may use our Security PGP Key available at `https://deployxa.com/security`.
- **Public Disclosure**: Please **do not** open public GitHub issues or disclose security vulnerabilities publicly until we have confirmed and released a fix.

### What to Include in Your Report
1. Detailed description of the vulnerability and its potential impact.
2. Step-by-step reproduction instructions or proof-of-concept script.
3. Affected components (`@deployxa/cli`, `deployxa-engine`, `deployxa-dashboard`, `docs.deployxa.com`).
4. Any suggested remediations or mitigations.

### Response SLAs
- **Initial Acknowledgment**: Within 24 hours.
- **Triage & Risk Assessment**: Within 48 hours.
- **Fix Release**: Critical security vulnerabilities will be patched within 7 days.

Thank you for helping keep Deployxa and our open-source developer community secure!
